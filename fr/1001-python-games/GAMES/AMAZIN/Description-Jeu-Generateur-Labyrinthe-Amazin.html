Amazin (Maze Generator)

Description

The game is a maze generator that creates a unique maze each time with a single correct path. The player defines the dimensions of the maze, and the program builds it taking into account the specified parameters.

  • Features:
    • The player enters the width and height of the maze.
    • The program guarantees a unique path through the maze.
    • The maze is displayed on the screen.

Step-by-step implementation instructions:

1. Game Initialization

  • Ask the player to enter the dimensions of the maze (width and height).
  • Check that the entered dimensions are valid (e.g., greater than one).

2. Main maze generation logic

  • Create a matrix of the specified size, representing a grid of cells.
  • Use a maze generation algorithm, for example, *recursive depth-first search (DFS) algorithm*:
    • Start from a random initial cell.
    • Move to adjacent cells, removing walls between the current cell and the next.
    • If all neighbors have been visited, return to the previous cell and continue.
    • End the process when all cells have been visited.
  • Guarantee a unique path through the maze.

3. Maze display

  • Use symbols for display:
    • +, -, | for walls.
    • Spaces for passages.
  • Display the generated maze as text.

4. Additional features

  • Possibility to define a predefined size (e.g., 10x10) if the user has entered invalid data.
  • Warning about excessively large maze sizes to avoid memory overload.
